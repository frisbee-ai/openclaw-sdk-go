Modify pkg/protocol/fuzz_test.go to add JSON round-trip correctness assertions to all 7 Fuzz* functions (D-05).

**Round-trip pattern for each Fuzz* function:**
```
1. Unmarshal input to frame struct (original)
2. Marshal frame back to JSON bytes
3. Unmarshal marshaled bytes to new frame struct (round-tripped)
4. Compare key fields: ID, Method, Action (for RequestFrame), Ok, Payload (for ResponseFrame), Event, Payload (for EventFrame)
5. On mismatch: t.Errorf (NOT panic -- fuzz test continues)
```

**For FuzzValidateRequestFrame:**
- After parsing, do: json.Unmarshal(data, &original) -> json.Marshal -> json.Unmarshal -> compare original.ID, original.Method, original.Params
- t.Errorf on mismatch: "RequestFrame round-trip mismatch: ID=%q want=%q"

**For FuzzValidateResponseFrame:**
- Compare original.Ok, original.ID, original.Error.Code if present, original.Payload

**For FuzzValidateEventFrame:**
- Compare original.Event, original.Payload, original.Seq

**For FuzzFrameType:**
- Compare FrameType value before and after round-trip (ft.String() -> FrameType() -> IsValid())

**For FuzzRequestID:**
- Compare frame.ID after round-trip

**For FuzzMethod:**
- Compare frame.Method after round-trip

**For FuzzEventType:**
- Compare frame.Event after round-trip

**For FuzzJSONMalformed and FuzzLargeInput and FuzzSpecialCharacters:**
- Apply round-trip to RequestFrame (most complete frame type)

**Key implementation notes:**
- Only compare fields that round-trip through JSON (ID, Method, Action, Payload). Internal-only fields (pointers, unexported) excluded.
- Use bytes.Equal for Payload comparison since it's json.RawMessage
- Continue fuzzing on mismatch (t.Errorf, not t.Fail)
- Keep the existing panic recovery (defer/recover) -- round-trip errors are reported but don't stop fuzzing
